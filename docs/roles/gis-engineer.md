## 岗位职责

1. 主导 **PC 端时空可视化系统** 前端开发，基于 Leaflet.js/OpenLayers 实现：
   - 多底图切换（现代/唐宋古地图/水墨底图）
   - 诗人完整行迹动态动画（三级时间轴控制 + 飞线动画 + 陆路/水路区分）
   - 多诗人同屏对比（2-10位叠加 + 颜色区分）
   - 诗词热力时序动态渲染
   - 交游网络可视化
2. 实现 **地图底层基础工具**：缩放、框选、距离测量、自定义围栏绘制、坐标工具
3. 设计并实现 **时空数据渲染管线**，满足单页 1000 点位 ≤1s 渲染的约束
4. 负责 **时空数据导出** 功能前端实现（Excel/CSV/GIS shp/高清 PNG ≥300DPI）
5. 与后端架构师协作设计 **GIS 接口规范**，参与 PostGIS 空间查询的性能优化
6. 参与交游概率算法、热力计算算法的工程化落地

## 关键挑战（面试必问）

1. **千点位渲染性能**：单页加载 1000+ 诗词点位并流畅渲染，如何优化？（考虑 Canvas 批量绘制、瓦片化、LOD 策略）
2. **飞线动画实现**：诗人一生路径动态播放，陆路/水路动线区分，暂停/倍速/拖动时间轴如何实现？
3. **热力时序更新**：拖动时间轴时热力图从唐代渐变到宋代，数据量 10万+ 级别，如何做到顺滑更新？
4. **多诗人对比叠加**：10位诗人的轨迹同屏显示，颜色区分，一键显隐，如何管理图层和数据状态？
