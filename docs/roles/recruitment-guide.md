## 面试评估工具包

每个 JD 文件中已内置：

1. **技术栈要求** — 硬性要求 vs 加分项，清晰区分
2. **关键挑战问题** — 针对本项目的核心场景问题（可用于技术面）
3. **面试评估要点** — 各维度的具体评估方式
4. **面试必问题** — 每个岗位 4 道核心场景题

### 面试建议流程

```
简历筛选     → 技术初面（PM + 候选人）    → 终面
(JD匹配度)     (关键挑战问题 + 代码能力)     (团队匹配 + 确认)
```

## 后端架构师 面试重点

建议让候选人现场做 **15 分钟架构设计**：

> **题目**："设计一个 80km 围栏查询方案，从数据库索引到 API 响应全链路，要求 ≤1s 返回 1000 点"
>
> **考察点**：
> - PostGIS GIST 空间索引理解
> - ST_DWithin 与 Haversine 取舍
> - 缓存策略（Redis Geo? 预计算?）
> - 数据量大时的分片方案

---

## GIS 工程师 面试重点

建议让候选人展示 **过往项目作品**：

> **考察点**：
> - 轨迹动画 / 热力图的真实项目经验（非 Demo 级别）
> - 千级点位渲染的性能优化方案
> - Canvas/WebGL 绘制能力
> - 对 Layer 管理的理解（多图层叠加/开关/层级）

---

## AI 算法工程师 面试重点

建议安排 **诗词知识 + NLP 能力双重考察**：

> **题目 1**（诗词基础）：给一首七律，请指出平仄格式和韵部
> **题目 2**（技术方案）：「白日依山尽」输入「依」字，如何从 265 万词库中 500ms 内返回对仗候选？
>
> **考察点**：
> - 是否真懂诗词格律（防伪）
> - 检索方案设计（倒排索引? 向量检索? 缓存层级?）
> - 工对/宽对/流水对的区分能力
