# 🔍 技术候选人筛选指南

> 供 PM 使用的简历评估标准
> 投资方转发简历后，我按此框架评估并给出录用建议

---

## 后端架构师 — 简历评估卡

| 评估维度 | 权重 | 关键信号（通过） | 危险信号（淘汰） |
|----------|------|-----------------|-----------------|
| Python 深度 | 25% | FastAPI/Flask/Django 生产项目，理解 async/await | 仅用 Python 写过脚本，无后端项目经验 |
| PostgreSQL 能力 | 25% | 写过复杂查询、了解索引原理、有空间数据经验 | 仅 CURD 级别，不知 EXPLAIN |
| 架构经验 | 25% | 主导过微服务拆分，描述得出服务间通信/数据一致性方案 | 一直在单体应用中，未接触过服务拆分 |
| GIS/时空数据 | 15% | 用过 PostGIS / Elasticsearch 时空索引 | 完全不了解 GIS 概念 |
| AI 集成 | 10% | 部署过模型推理服务 | 无 AI 相关经验 |

**初步结论建议：**
- 满足 4/5 项 → 安排面试
- 满足 3/5 且 Python+PG 强 → 可面试
- 不满 3/5 → 暂不推进

---

## GIS 工程师 — 简历评估卡

| 评估维度 | 权重 | 关键信号（通过） | 危险信号（淘汰） |
|----------|------|-----------------|-----------------|
| 地图引擎 | 30% | Leaflet/OpenLayers 真实项目，非 Demo | 仅用过百度/高德 API 做简单标注 |
| 可视化能力 | 25% | Canvas/WebGL/D3 做轨迹/热力/动画 | 无自定义渲染经验 |
| React 能力 | 20% | React+TS 生产项目 | 不会 React |
| 性能优化 | 15% | 有千级点位渲染优化经验 | 未考虑过渲染性能 |
| GIS 基础 | 10% | 理解 WGS84/投影/GeoJSON | 不理解坐标系概念 |

**重要：** GIS 工程师必须有 **作品展示** — 面试时要求看真实项目（非课堂/培训 Demo）。

## AI 算法工程师 — 简历评估卡

| 评估维度 | 权重 | 关键信号（通过） | 危险信号（淘汰） |
|----------|------|-----------------|-----------------|
| NLP 项目经验 | 30% | 文本生成/序列标注/分类真实项目 | 无 NLP 项目经验 |
| 诗词/NLP 交叉 | 20% | 有过古典诗词 NLP 项目 | 对诗词完全不了解 |
| 工程化能力 | 20% | 模型部署、性能优化（ONNX/量化）、FastAPI 封装 | 只有 Jupyter 经验，无服务化经验 |
| Python/框架 | 15% | Python 熟练 + PyTorch/Transformers | Python 不熟 |
| 诗词基础 | 15% | 了解平仄/押韵/对仗基础概念 | 完全不懂格律 |

**注意：** 这个岗位需要有 **诗词热情** — 能力可以培养，但没兴趣的人做不出好产品。

## 简历评估流程

```
投资方转发简历
    ↓
PM 按上述标准评估 → 输出评估意见（通过/待定/不通过）
    ↓
评估通过 → 设计技术面试题（从 JD 的"关键挑战"中选）
    ↓
安排面试（投资方组织，我出题 + 定标准）
    ↓
面试后 → 我给出录用评分和建议
    ↓
投资方发 Offer
```
