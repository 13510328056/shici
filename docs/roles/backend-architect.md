## 岗位职责

1. 负责平台整体后端架构设计，主导微服务拆分方案（7类微服务 + API Gateway + BFF 层），制定服务间通信、数据一致性、异常处理策略
2. 主导 **PostgreSQL + PostGIS** 时空数据库设计，包括古今地名映射库、诗人轨迹库、诗词多维特征库的三库逻辑模型与物理模型
3. 负责 **Elasticsearch/OpenSearch** 时空复合索引设计，支撑六维度组合检索 ≤500ms 的约束
4. 主导 **FastAPI** 开发规范制定，包括 RESTful API 风格、认证鉴权方案（JWT）、统一异常处理、分页规范
5. 负责 AI 服务与主业务服务的集成方案设计（AI 对仗/格律/文本生成为独立子服务）
6. 制定数据迁移、备份、灾备策略，确保90天全链路日志留存
7. 技术选型关键决策，参与 GIS / AI 方案的技术评审
8. 指导后端团队（2-3人）的日常开发与代码评审

## 技术要求

### 硬性要求
- ✅ **5年以上**后端开发经验，**2年以上**架构设计经验
- ✅ 精通 **Python**，有 **FastAPI** 或类似异步框架生产项目经验
- ✅ 精通 **PostgreSQL**，有 **PostGIS** 扩展实际使用经验（空间索引、GIS 函数等）
- ✅ 熟悉微服务架构设计，有 Docker/K8s 容器化部署经验
- ✅ 理解 RESTful API 设计原则，熟悉 JWT/OAuth2 认证机制

### 加分项
- ⭐ 有 **Elasticsearch/OpenSearch** 时空索引设计经验
- ⭐ 有 GIS 相关项目经验（Leaflet/OpenLayers 服务端支撑）
- ⭐ 有 AI/ML 服务集成经验（模型部署、推理管线）
- ⭐ 熟悉《中国历史地名大辞典》或数字人文项目者优先
- ⭐ 有开源项目贡献经历

## 关键挑战（面试必问）

1. **80km 时空围栏查询**：如何设计索引支撑跨 6 万地名、百万级点位的围栏查询性能？
2. **AI 服务 ≤500ms 约束**：对仗推荐需要毫秒级响应，如何设计缓存/预计算架构？
3. **三源数据整合**：来自不同数据源（历史地名大辞典/唐宋文学编年/诗词语料）的数据如何 ETL 并建立关联？
4. **多终端同源**：PC/App/小程序/AR 共用同一套 API，BFF 层如何裁剪数据？
