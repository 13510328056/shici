## 岗位职责

### App 端（React Native）
1. 基于 **React Native** 开发诗词文化互动平台 App（Android + iOS）
2. 实现地图导航功能（集成地图 SDK，展示诗词点位、主题路线）
3. 实现基础诗词检索与展示功能
4. 实现 **文旅交互功能**：景区扫码讲解、诗词打卡创作、社交分享
5. 实现 **AR 基础体验**（调用摄像头 + 定位触发 AR 内容展示）
6. 实现 AI 创作工具（对仗推荐、格律校验的移动端适配）

### 小程序端（Taro）
7. 基于 **Taro** 开发微信小程序版本
8. 保留轻量版核心功能：景点扫码讲解、基础地图导航、诗词检索

## 技术要求

### 硬性要求
- ✅ **3年以上**移动端开发经验
- ✅ 精通 **React Native**，有上架 App Store / 安卓应用商店的正式产品
- ✅ 精通 **TypeScript/JavaScript**
- ✅ 有移动端地图组件集成经验（高德/百度/Google Maps SDK 等）
- ✅ 有 **Taro** 或类似多端框架（uni-app）的小程序项目经验
- ✅ 了解移动端性能优化（列表虚拟化、图片缓存、包体积控制）

### 加分项
- ⭐ 有 WebView ↔ Native 桥接通信开发经验
- ⭐ 有 AR 集成经验（ARKit/ARCore/ViroReact）
- ⭐ 有音频播放/录制功能开发经验（诗词配乐朗诵场景）
- ⭐ 同时有 Android 原生（Kotlin）和 iOS 原生（Swift）开发能力
- ⭐ 有微信小程序的审核/发布经验

## 关键挑战（面试必问）

1. **多端复用策略**：PC 用 Leaflet/OpenLayers，App 用地图 SDK，如何设计共享逻辑层？
2. **AR 轻量化方案**：手机端 AR 体验如何在非旗舰机上流畅运行？实现方案是什么？
3. **离线支持**：景区网络不稳定情况下如何保障扫码讲解等基础功能可用？
4. **包体积控制**：React Native + 地图 SDK + AR 库 → 如何控制 App 包体积？
