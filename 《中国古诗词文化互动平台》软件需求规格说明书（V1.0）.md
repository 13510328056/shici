# 正文

## 1 引言

### 1.1 项目背景

当前古典诗词数字化产品在时空关联、AI 创作辅助和文旅沉浸交互方面存在明显短板：一是诗词文本与地理时空信息割裂，缺少完整古今地名映射和诗人行迹量化分析；二是创作辅助工具功能单一，仅支持基础对仗查询，缺乏完整格律校验、意境融合和多风格改写能力；三是文旅场景数字化程度浅，缺少 AR 实景、时空路线和在地创作一体化交互体系。
平台整合历史地理文献、唐宋文学编年数据与千万级诗词语料，融合 GIS 时空可视化、SLAM 增强现实、NLP、检索与可视化技术，构建学术研究、文旅体验与大众诗词创作三大场景一体化平台，补强当前产品在时空分析、AI 创作辅助与沉浸交互方面的能力。

### 1.2 项目建设目标

1. **数据目标**：构建标准化、可溯源、可拓展的古今地名、诗人轨迹与诗词多维特征三大数据库，确保古今地名精准映射、诗人活动时空全记录以及诗词六维度结构化标注；
2. **学术目标**：提供量化时空分析工具，实现诗人轨迹动态可视化、文人交游概率计算及诗词地域分布时序热力分析，支撑数字人文学术研究；
3. **文旅目标**：面向景区与游客提供诗词主题导航、AR 实景复原和诗词打卡创作等沉浸式交互功能，提升诗词文化深度游体验；
4. **大众普及目标**：构建全链路 AI 诗词创作辅助工具链，降低格律、意境与对仗创作门槛，推动传统文化大众化传承；
5. **技术目标**：采用分层微服务架构，多终端统一业务逻辑，具备数据扩容、AI 能力迭代与第三方渠道对接的扩展能力。

### 1.3 文档适用范围

1. 内部：产品设计、前后端研发、GIS 开发、AI 算法工程师、测试工程师、UI/UX 设计师、项目管理；
2. 外部：文史专家评审、文旅合作单位、景区运营方、第三方集成服务商；
3. 下游产出：系统概要设计、详细设计、数据库设计、接口文档、测试方案、验收标准。

### 1.4 术语、缩写与定义

1. WGS84 (EPSG:4326)：全球统一地理坐标系，平台所有地名坐标标准；
2. SLAM：即时定位与地图构建，用于 AR 实景空间叠加；
3. 时空围栏：以诗人活动点位为中心，80km 半径圆形地理范围；
4. 时间容错窗口：±1 年，抵消古代纪年、史料记载误差；
5. 交游概率模型：量化两位诗人时空重叠相遇可能性的计算公式；
6. 时空复合索引：基于开源搜索引擎构建的 “空间 + 时间 + 主题” 联合检索索引；
7. 多维特征标注：诗词统一标注维度：地理、时间、人物、意象、意境、用典；
8. 对仗词库：基于 39.3 万首律诗、1.5 万首排律提取的 265 万组标准化对仗词汇；
9. AR 触发点：景区内置定位识别点位，用于唤醒历史场景、数字诗人交互。

### 1.5 参考资料

1. 《古诗词地名可视化研究》Word 完整版、PDF 精简版原始研究文档；
2. 《中国历史地名大辞典（2017 增订本）》；
3. 王兆鹏团队唐宋文学编年系地数据集；
4. 国家民政部地名信息库、省级地名文化遗产数据集；
5. 开源 GIS、Web AR 技术规范，如 Leaflet.js、OpenLayers、WebXR/AR.js；
6. 开源搜索引擎时空索引技术白皮书。

### 1.6 需求优先级定义

* **P0（一级核心，上线必达）**：底层三库、基础地图、诗人动态轨迹、多维检索、基础对仗 / 意境 AI 工具、小程序基础导航；
* **P1（二级重要，首期上线）**：文人交游概率计算、诗词热力时空分析、格律校验、App 完整功能、基础 AR 场景、景区扫码讲解；
* **P2（三级拓展，一期迭代）**：多意境融合、诗词仿写改写、学术图表导出、用户打卡分享、简易文创数据统计；
* **P3（中长期迭代，二期及以后）**：全朝代数据扩容、众包标注、大模型深度融合、跨平台渠道对接。

## 2 总体业务与用户需求

### 2.1 用户分层与角色画像

1. **学术研究者（PC 端核心用户）**
人群：高校文史教师、古代文学 / 历史地理研究生、文史研究院研究员；
设备：PC 网页端，少量移动端查阅资料；
2. **文旅游客（App / 小程序核心用户）**
人群：诗词主题景区游客、古风爱好者、短途文化旅行者；
设备：手机 App、微信小程序、AR 智能眼镜；
3. **诗词学习爱好者（全终端通用）**
人群：中小学生、诗词业余学习者、格律创作爱好者；
设备：手机、PC、平板；
4. **文旅运营管理员（后台管理端）**
人群：景区数字化负责人、文旅项目运营人员；
设备：PC 管理后台、移动端简易管理面板。

### 2.2 各角色核心业务诉求

1. 学术研究者：批量时空数据导出、多诗人轨迹对比、文人交游量化分析、时序热力统计、可导出学术图表、精准古今地名溯源；
2. 文旅游客：一键诗词主题路线、实景 AR 沉浸式体验、景点诗词音频讲解、线下打卡 AI 写诗、作品社交分享；
3. 诗词爱好者：全维度诗词检索、智能对仗推荐、格律自动校正、意境匹配写作框架、诗词 AI 解析与优化；
4. 文旅管理员：景区诗词点位管理、AR 内容后台配置、用户打卡数据统计、创作热力图数据导出。

### 2.3 平台整体业务边界（明确不包含范围，规避范围蔓延）

1. 不包含完整诗词在线直播、线上诗词课程付费教学系统；
2. 不包含线下实体文创产品生产、电商售卖完整交易链路；
3. 不包含历史地图原始测绘、全新古籍数字化扫描业务；
4. 不支持第三方付费广告投放、流量变现相关模块；
5. 不提供完整词牌、曲谱音频演唱生成功能（仅提供诗词文字朗诵音频）。

### 2.4 产品定位锚点

本平台的核心定位是“**诗词时空可视化学术工具 + 文旅增值体验**”。在V2.0中，明确产品的核心发力点：

* **MVP核心：** 以**学术工具**（PC端）为突破口，优先满足学术研究者对**批量时空数据导出、多诗人轨迹对比、文人交游量化分析、时序热力统计、可导出学术图表、精准古今地名溯源**的核心需求。这是V2.0的首要目标，旨在快速验证核心价值并建立用户基础。
* **文旅增值：** 在学术工具稳定基础上，通过**诗词主题导航、AR 实景复原、诗词打卡创作**等功能，为文旅游客提供沉浸式体验，赋能诗词文化深度游，探索商业化路径。

## 3 系统总体架构需求

### 3.1 架构总体设计原则

1. 分层解耦：数据、服务、应用三层完全隔离，单一层变更不影响其他层级；
2. 标准统一：地理坐标、数据字段、API 入参出参、标注规则全局标准化；
3. 多端同源：PC、App、小程序、AR 眼镜共用同一套后台服务，业务逻辑无差异化；
4. 可扩展设计：预留朝代数据扩容、大模型接入、第三方文旅平台对接接口；
5. 可追溯：所有地名、诗词、轨迹数据标注来源留痕，支持溯源查阅。

### 3.2 数据层：三源一体标准化数据库

#### 3.2.1 古今地名映射数据库

1. 数据源约束：主数据源为《中国历史地名大辞典（2017 增订本）》，辅以《元和郡县图志》《读史方舆纪要》、国家民政部官方地名库；
2. 数据规模：不少于 60000 条古今地名对照词条；
3. 强制字段规范：古地名、现代标准地名、WGS84 经纬度、省级 / 市级 / 区县行政区划、地名沿革时间线、关联诗词 ID 索引；
4. 歧义地名处理规则：针对 “紫台”“大梁”“楼兰” 等多义地名，绑定诗词语境判定逻辑，根据诗文创作时代、上下文区分地理指向；
5. 关联关系存储：记录地标空间从属 / 相邻关系（长江 — 黄鹤楼、赤壁），存储地名历史名称变更时序（金陵、武昌等典型案例）。

#### 3.2.2 诗人生平轨迹时空数据库

1. 基础数据源：王兆鹏唐宋文学编年系地数据，预留扩展字段支持先秦、汉魏、元明清诗人接入；
2. 数据表固定三大分区：

   * 诗人基础信息：唯一 ID、姓名、生卒年月（精确至月份）、朝代、身份标签；
   * 时空轨迹记录：事件时间戳、古地名、现地名、经纬度、活动类型枚举（出生 / 科举 / 仕宦 / 贬谪 / 游览 / 创作）、原地停留时长；
   * 作品关联：绑定创作作品唯一 ID、诗文标题、标准化创作背景；
3. 拓展存储：诗人交游关联中间表，存储同空间同时段诗人匹配记录，支撑交游概率计算。

#### 3.2.3 诗词多维特征标注数据库

全库诗词强制六维度结构化标注，标注内容可检索、可统计、可用于 AI 匹配：

1. 地理特征：创作地点、诗文描写地点；
2. 时间特征：创作年份、描写四季、二十四节气、传统节日；
3. 人物特征：诗文描写人物、典故历史人物；
4. 物品意象特征：自然景物、建筑、器物；
5. 意境特征：送别、思乡、边塞、田园、怀古、闺怨等分类标签；
6. 用典特征：典故名称、典故出处、典故指代对象。

### 3.3 服务层：微服务 API 集群需求

后台拆分为 7 类独立微服务，提供标准化 RESTful 接口，支持鉴权、分页、异常统一返回；AI 诗词辅助服务进一步拆分为 3 个子服务，便于能力隔离与弹性扩展。统一入口与端侧适配层补充如下：

1. Auth 服务（新增）：统一用户认证、授权、JWT Token 管理；
2. 地名映射服务：古今地名模糊查询、坐标双向转换、地名沿革时序查询；
3. 时空可视化计算服务：轨迹动画渲染、热力图生成、文人交游概率批量运算、时空统计计算；
4. 多维检索服务：多维度组合检索、文本相似度计算、季节 / 节气模糊匹配；
5. 文旅推荐服务：基于用户定位、诗词主题智能生成游览路线、景点诗词推送；
6. AI 诗词辅助服务：内部拆分为 AI 对仗推荐、AI 格律校验、AI 文本生成三个子服务；
7. 数据导出服务：时空数据表、热力图表、诗人轨迹 GIS 文件批量导出；

AI 诗词辅助子服务：
* AI 对仗推荐服务：对仗推荐、上下文情感识别；
* AI 格律校验服务：格律校验、押韵校正；
* AI 文本生成服务：意境框架生成、诗词解析、仿写改写、多意境融合；

* API Gateway（新增）：统一入口、路由、限流、鉴权转发；
* BFF 层（新增）：为 PC、App、小程序、AR 各设 BFF 服务，做数据裁剪、协议适配、端级缓存。

### 3.4 应用层：多终端产品适配需求

1. PC 端 Web 专业系统（P0 级）
面向学术用户，开放完整图层自定义、批量数据导出、多诗人对比、学术图表生成、GIS 文件下载；
2. 移动端 App（P0 级）
面向游客与爱好者，优先提供地图导航、基础诗词检索、主题推荐与创作引导；
3. 微信小程序（P0 级轻量化）
无需安装，仅保留景点扫码诗词讲解、简易诗词地图导航、基础诗词检索；
4. AR 眼镜配套应用（P2 级）
面向硬件创新场景，构建基于开放 AR 标准与 SLAM 的户外实景叠加和交互演示能力；
5. 管理后台（P1 级）
面向文旅运营，提供景点点位管理、AR 内容配置、用户创作数据统计。

## 4 核心功能详细需求（含两大重点扩充模块）

### 4.1 诗词时空可视化综合模块（重点扩充，P0/P1 核心）

本模块为平台差异化核心能力，统一基于开源 GIS 底层地图引擎，如 Leaflet.js、OpenLayers，所有坐标强制采用 WGS84 标准。

#### 4.1.1 基础地图底层支撑能力（P0）

1. 多底图切换：现代标准地图、唐代古地图、宋代古地图、水墨诗词极简底图；
2. 图层可视化管理：支持手动开关地名、诗人轨迹、诗词点位、热力、交游标记五大图层；
3. 地图基础工具：无级缩放、框选地理范围、距离测量、自定义点位标记、手动绘制地理围栏；
4. 坐标工具：支持手动录入经纬度定位、批量导入坐标点位批量渲染。

#### 4.1.2 诗人完整行迹动态可视化系统（P0）

1. 三级时间轴控制：滑块拖动、年份快捷按钮、精确年月手动输入；支持单年份、时间段、全生命周期三种查看模式；
2. 动态飞线动画：可播放 / 暂停 / 倍速播放诗人完整游历路线，自动区分陆路、水路动线；
3. 标准化事件标记体系（颜色图标固定，不可自定义）：

   * 蓝色圆形：出生地、长期居住地；
   * 绿色方形：诗词原创地点；
   * 红色三角形：贬谪、流放地点；
   * 黄色星形：文人雅集、交游地点；
点击标记弹窗展示：发生时间、事件类型、停留时长、关联诗文列表、创作背景；
4. 单诗人统计面板：右侧实时展示游历城市总数、各地创作诗词数量、仕宦 / 贬谪 / 游览分段时长统计。

#### 4.1.3 多图层时空对比分析功能（P1）

1. 多诗人同屏对比：支持同时加载 2\~10 位诗人轨迹，不同颜色飞线区分，一键隐藏 / 显示单人行迹；
2. 跨朝代图层对比：分别加载唐代、宋代诗人图层，直观对比两代诗词创作地域热点差异；
3. 季节 / 节气筛选：筛选春、夏、秋、冬、清明、中秋等时间标签，生成分季节创作点位分布图；
4. 流域分区筛选：一键筛选长江、黄河、边塞、江南等地理分区，分区统计诗人、作品数量。

#### 4.1.4 文人时空交游交集量化分析系统（P1）

1. 算法参数固化（不可动态修改，统一标准）：

   * 时空围栏半径：80km；
   * 时间容错窗口：±1 年；
   * 交游概率计算公式：$P = \\frac{重叠时间}{两人各自总停留时间} \\times \\frac{重叠活动面积}{两人各自活动总面积}$
2. 可视化输出约束：

   * 全域交游热力图：色彩梯度代表相遇概率高低；
   * 重叠区域点击弹窗：交集诗人名单、标准化相遇概率、同期留存诗文、双人行迹快速对比入口；
3. 导出能力：支持诗人交游关联网络图高清图片、CSV 关系数据表导出。

#### 4.1.5 诗词地域分布热力时空分析（P1）

1. 全局诗词热力渲染：依据一地收录诗词总量渲染热度层级；
2. 时序动态热力：拖动时间轴，热力图随年份动态刷新，展示文学中心地域迁移；
3. 多维度分区统计：按省份、历史道 / 州、自然流域自动统计诗词总量；
4. 主题热力筛选：单独渲染边塞、田园、怀古、送别类诗词专属热力分布图。

#### 4.1.6 时空数据导出与学术图表生成（P2）

1. 导出文件格式：Excel、CSV、GIS shp 标准文件、高清 PNG 地图截图；
可导出内容：地名坐标全量表、诗人轨迹明细、诗词点位、交游概率数据；
2. 自动生成统计图表：时间分布折线图、地域诗词柱状图、诗人停留时长统计图；
3. 图片分辨率：地图、热力图导出最低 300DPI，满足学术论文印刷标准。

### 4.2 多维诗词聚合检索模块（P0）

#### 4.2.1 六大维度组合检索能力

支持空间、时间、人物、物品意象、意境、用典自由交叉检索；支持 “早春”“重阳” 等模糊季节、节日匹配检索。

#### 4.2.2 检索算法与性能约束

1. 单维度检索：数据库字段精准匹配；
2. 多维度组合检索：向量空间模型计算诗文匹配相似度，按相关性降序排序；
3. 性能约束：采用开源搜索引擎时空-主题复合索引，10 万条数据内检索响应≤500ms。

#### 4.2.3 检索结果三形态展示

地图点位视图、诗文列表视图、多诗横向对比视图；支持同一地点不同时节、不同诗人同题材作品对照赏析。

### 4.3 诗词沉浸式文旅交互模块（P1）

#### 4.3.1 诗词主题智能导航路线规划

自动读取用户 GPS 定位，提供山水田园、边塞豪情、怀古咏史、三苏文化等预设主题路线；靠近诗词景点自动弹窗推送诗词、配乐朗诵音频。

#### 4.3.2 AR 实景诗人沉浸式交互体验

景区预设 AR 触发点位，3D 模型复原古代场景；AI 数字诗人实景对话讲解生平创作故事；AR 场景同步播放对应诗词配乐朗诵。

#### 4.3.3 景区标准化诗词文化展示配套

景点二维码扫码讲解、电子诗词循环大屏、地面光影诗词长廊、声光意境氛围还原。

#### 4.3.4 文旅诗词打卡与原创分享系统

景点现场创作引导、AI 智能评分、社交平台一键分享、全域用户创作热力地图、打卡数据后台统计导出。

### 4.4 AI 全链路诗词创作辅助模块（重点扩充，P1/P2 分级）

完整覆盖诗词创作全流程：选词对仗→框架生成→格律校验→全文解析→改写仿写→多意境融合。根据评审意见，本模块优先级从 P0 调整为 P1，分层级迭代上线：

* P1（二级重要，首期上线）：百万级对仗词汇智能推荐、多维度意境智能匹配创作、格律校验、押韵校正；
* P2（三级拓展，一期迭代）：诗词全文 AI 解析与诊断优化、多意境融合框架生成、诗词仿写、扩写、改写 AI 辅助。

#### 4.4.1 百万级对仗词汇智能推荐子系统（P1）

1. 底层语料约束：基于 39.3 万首律诗、1.5 万首排律，提取 265 万组标准化对仗词汇；
2. 核心逻辑：词根对仗链拆解、上下文情感 / 意境识别、动态过滤违和词汇；
3. 交互规范：输入框自动高亮对仗位置，输入文字实时下拉推荐；点击词汇展开完整对仗延伸链，附带历代经典诗句范例；区分工对、宽对、流水对，标注适配体裁（五律 / 七律 / 排律）。

#### 4.4.2 多维度意境智能匹配创作工具（P1）

1. 意境分类模型：内置送别、思乡、边塞、田园、怀古、登临、闺怨标准意境标签库；
2. 多维条件匹配：用户自选地点、季节、节气、情感意境，系统匹配同场景经典诗词；
3. 双层创作框架：入门极简模板、进阶完整格律框架，每联自动推荐适配意象词汇；
4. 场景化写作指引：针对山川、古都、边塞、园林输出专属景物描写、抒情技巧提示。

#### 4.4.3 格律校验、押韵校正智能工具（P1）

1. 全体裁覆盖：绝句、律诗、古风、主流词牌自动校验；
2. 双韵库支持：平水韵、中华新韵自由切换；
3. 可视化错误标注：平仄失调、出韵、失粘失对文字标红悬浮，提供替换修正方案。

#### 4.4.4 诗词全文 AI 解析与诊断优化（P2）

1. 自动解析报告：提取意象、情感、主题、修辞手法、典故；
2. 意境相似度计算：对比用户诗作与名家诗词匹配度，输出优劣分析；
3. 三层优化方案：轻度字词修改、中度句式改写、整体意境重构三套方案可选。

#### 4.4.5 多意境融合诗词框架生成引擎（P2）

1. 多诗要素提取：支持选取 2\~5 首经典诗词，拆解意象、情感、景物要素；
2. 融合算法：自动剔除冲突意象，生成全新写作框架；
3. 风格约束：可选唐诗雄浑、宋词婉约、田园清淡、边塞豪放等风格限定输出。

#### 4.4.6 诗词仿写、扩写、改写 AI 辅助（P2）

1. 同风格仿写：输入古诗，生成同体裁、同意境、不同意象范文；
2. 短句扩写：两句短句扩展为完整律诗 / 词；
3. 体裁互转：绝句↔律诗、古诗转词、白话文字转古典诗词；
4. 视角改写：切换游子、隐士、征人等多视角重写全诗。

## 6 非功能性需求（评审核心校验项）

### 6.1 性能需求

1. 地图渲染：单页加载 1000 条诗词点位，渲染完成≤1s；
2. 多维检索：10 万条诗词库，复合检索响应≤500ms；
3. AI 工具单次请求（修订）：
   * 快查询（对仗推荐、格律预检）响应≤500ms；
   * 生成类操作（意境框架、仿写改写等）采用异步+流式返回，首 token≤800ms；
4. 并发支撑：首期支持 500 人同时在线，预留扩容至 2000 并发的架构能力；
5. 时空热力批量计算：单区域热力生成≤3s（针对≤10000 点）/≤10s（针对>10000 点）。

1. PC 端：兼容 Chrome、Edge、Firefox 主流浏览器，最低兼容 Windows10；
2. 移动端 App：Android 8.0 及以上、iOS 13.0 及以上；
3. 小程序：微信基础库 2.20 以上；
4. AR 眼镜：适配支持开放 AR 标准设备。

### 6.3 安全与权限需求

1. 数据安全：用户原创诗词、自定义标注数据加密存储；地名、诗词核心数据库只读隔离；
2. 三级权限体系：普通游客、文史学者、文旅管理员、超级管理员；不同角色功能可见范围隔离；
3. 接口安全：所有 API 接入统一 Token 鉴权，防重复请求、恶意批量爬取；
4. 内容安全：用户原创诗词内置敏感词过滤，违规内容拦截不入库。

### 6.4 易用性需求

1. 学术端：内置操作手册、时空分析功能分步引导；
2. 游客端：简化操作路径，AR、导航一键启动，无复杂配置；
3. AI 创作工具：新手弹窗引导，术语简易注释，格律错误直观标注。

### 6.5 可扩展性需求

1. 数据扩展：数据库预留朝代拓展字段，支持新增元明清、先秦诗词数据；
2. AI 扩展：预留大模型接入标准接口，可替换 / 升级 NLP 模型；
3. 渠道扩展：预留第三方文旅平台、校园系统对接 API。

### 6.6 可维护性需求

1. 日志留存：接口访问、地图计算、AI 调用全链路日志留存≥90 天；

3. 数据修正：地名、诗词标注支持后台批量修正，修改记录留痕可追溯。

## 9 附录

### 9.1 需求整合说明

本需求完整融合两份原始研究文档全部数据标准、技术方案、文旅场景、AI 工具逻辑；针对**诗词时空可视化**增加多层对比、时序热力、学术导出全流程能力；针对**AI 诗词辅助**扩充格律校验、多意境融合、仿写改写完整工具链，兼顾学术专业深度与大众文旅轻量化使用需求，所有功能指标、算法参数、数据来源均有原始文档支撑，无虚构需求。

### 9.2 功能优先级分级清单

* P0（一级核心，上线必达）：底层三库、基础地图、诗人动态轨迹、多维检索、基础对仗 / 意境 AI 工具、**App 基础版**、小程序基础导航；
* P1（二级重要，首期上线）：文人交游概率计算、诗词热力时空分析、格律校验、**App 完整功能**、**社交分享**、基础 AR 场景（手机端）、景区扫码讲解；
* P2（三级拓展，一期迭代）：多意境融合、诗词仿写改写、学术图表导出、**AR 眼镜正式版**、简易文创数据统计；
* P3（中长期迭代，二期及以后）：全朝代数据扩容、众包标注、大模型深度融合、跨平台渠道对接。

### 9.3 核心算法参数固化标准

1. 地理围栏半径：80km；
2. 时间容错窗口：±1 年；
3. 地理坐标系：WGS84（EPSG:4326）；
4. 对仗词库规模：265 万组，源语料 39.3 万首律诗、1.5 万首排律；
5. 检索索引：开源时空主题检索引擎（如 Apache Lucene、OpenSearch）构建的复合索引；
6. 交游概率计算公式：$P = \frac{重叠时间}{T_1+T_2-重叠时间} \times \frac{重叠活动面积}{A_1+A_2-重叠活动面积}$，确保 $0 \leq P \leq 1$。
